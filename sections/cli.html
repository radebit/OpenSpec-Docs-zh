openspec CLI 提供项目设置、验证、状态检查和管理的终端命令

分类命令用途
设置init、update初始化和更新项目中的 OpenSpec
工作区（Beta）workspace setup、workspace list、workspace link、workspace relink、workspace doctor、workspace open跨仓库或文件夹的协作工作区规划

配置config查看和修改设置

初始化与更新

openspec init 在项目中初始化 OpenSpec
语法
openspec init [path] [options]
选项描述
--tools <list>非交互式配置 AI 工具。使用 all、none 或逗号分隔的列表
--force自动清理旧文件，无需提示
--profile <profile>覆盖此次初始化的全局 profile（core 或 custom）
示例
# 交互式初始化
openspec init

# 非交互式：配置 Claude 和 Cursor
openspec init --tools claude,cursor

# 配置所有支持的工具
openspec init --tools all

# 覆盖 profile
openspec init --profile core
openspec update 升级 CLI 后更新 OpenSpec 指令文件
示例
# npm 升级后更新指令文件
npm update @fission-ai/openspec
openspec update

工作区命令 Beta

⚠️ 处于积极开发中：工作区相关命令尚未正式发布。请不要基于此命令面构建外部自动化、集成或长期工作流——命令行为、状态文件和 JSON 输出随时可能变更。

协作工作区（Coordination Workspace）是用于规划跨多仓库/文件夹工作的久久规划家园。工作区只是「让 OpenSpec 看到哪些仓库/文件夹」，并不代表变更承诺。

openspec workspace setup 创建工作区并链接到至少一个已存在的仓库/文件夹
选项描述
--name <name>工作区名（kebab-case）
--link <path>链接已存在的仓库/文件夹，从文件夹名推断链接名
--link <name>=<path>显式指定链接名
--opener <id>偏好打开器：codex、claude、github-copilot、editor
--no-interactive禁用交互（需 --name 和至少一个 --link）
--jsonJSON 输出（需配合 --no-interactive）
示例
# 交互式向导
openspec workspace setup

# 自动化场景
openspec workspace setup --no-interactive --name platform \
  --link /repos/api --link web=/repos/web

# 带偏好打开器
openspec workspace setup --no-interactive --name platform \
  --link /repos/api --opener codex
openspec workspace list / ls 列出本机已知工作区及其链接
示例
openspec workspace list
openspec workspace ls
openspec workspace list --json
openspec workspace link 向工作区登记一个已存在的仓库/文件夹
示例
# 从文件夹名自动推断链接名
openspec workspace link /repos/api

# 显式指定链接名
openspec workspace link api-service /repos/api

# 指定目标工作区
openspec workspace link --workspace platform /repos/platform/apps/checkout
openspec workspace relink 修复或更改已有链接的本地路径
示例
openspec workspace relink api-service /new/path/to/api
openspec workspace doctor 检查当前机器上工作区的解析情况
示例
openspec workspace doctor
openspec workspace doctor --workspace platform

显示工作区位置、规划路径、已链接仓库/文件夹、缺失路径及修复建议；仅报告问题，不自动修复。

openspec workspace open 通过偏好打开器启动工作区
选项描述
--workspace <name>工作区名称（位置参数的别名）
--agent <tool>本次会话覆盖：codex、claude、github-copilot
--editor以 VS Code 编辑器模式打开多根工作区文件
--no-interactive禁用工作区和打开器选择提示
示例
openspec workspace open
openspec workspace open platform
openspec workspace open platform --agent github-copilot
openspec workspace open --editor
💡
OpenSpec 会自动维护工作区根下的 <workspace-name>.code-workspace 多根工作区文件（仅本机），默认已加入 .gitignore。跨路径（Windows/WSL2等）不做自动转换，请使用运行时匹配的路径格式。

浏览命令

openspec list 列出项目中的变更或规范
示例
# 列出所有活跃变更
openspec list

# 列出所有规范
openspec list --specs

# JSON 输出（用于脚本）
openspec list --json
openspec show 显示变更或规范的详情
示例
# 显示特定变更
openspec show add-dark-mode

# JSON 输出用于解析
openspec show add-dark-mode --json

验证命令

openspec validate 验证变更和规范的结构问题
示例
# 验证特定变更
openspec validate add-dark-mode

# 验证所有变更
openspec validate --changes

# 验证所有内容并输出 JSON（用于 CI）
openspec validate --all --json

# 严格模式
openspec validate --all --strict --concurrency 12

生命周期命令

openspec archive 归档已完成的变更并将 Delta 规范合并到主规范
示例
# 归档特定变更
openspec archive add-dark-mode

# 无提示归档（CI/脚本）
openspec archive add-dark-mode --yes

# 归档不影响规范的工具变更
openspec archive update-ci-config --skip-specs

工作流命令

openspec status 显示变更的产物完成状态
示例
openspec status --change add-dark-mode

# 输出:
# Change: add-dark-mode
# Schema: spec-driven
# Progress: 2/4 artifacts complete
#
# [x] proposal
# [ ] design
# [x] specs
# [-] tasks (blocked by: design)

Schema 命令

openspec schema init 创建新的项目本地 Schema
示例
# 交互式创建 Schema
openspec schema init research-first

# 非交互式
openspec schema init rapid \
  --description "快速迭代工作流" \
  --artifacts "proposal,tasks" \
  --default
openspec schema fork 复制现有 Schema 到项目以进行自定义
示例
# Fork 内置的 spec-driven schema
openspec schema fork spec-driven my-workflow
🔢
Schema 优先级
1. 项目级：openspec/schemas/<name>/
2. 用户级：~/.local/share/openspec/schemas/<name>/
3. 包级：内置 Schema

配置命令

openspec config 查看和修改全局 OpenSpec 配置
示例
# 列出所有设置
openspec config list

# 获取特定值
openspec config get telemetry.enabled

# 设置值
openspec config set telemetry.enabled false

# 配置工作流 profile
openspec config profile

# 快速切换到 core profile
openspec config profile core

# 在编辑器中编辑
openspec config edit

环境变量

变量描述
OPENSPEC_CONCURRENCY批量验证的默认并发数（默认: 6）
EDITOR 或 VISUALopenspec config edit 使用的编辑器
NO_COLOR设置时禁用彩色输出
OPENSPEC_TELEMETRY=0禁用遥测数据收集
DO_NOT_TRACK=1全局禁用追踪
